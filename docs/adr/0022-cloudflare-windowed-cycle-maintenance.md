# Run Cycle maintenance in a bounded Cloudflare schedule window

Church Work runs Rollover Maintenance from the existing web Worker every 15 minutes from Sunday 10:00 UTC through Monday 12:00 UTC, covering every Church-local Monday boundary from UTC+14 through UTC-12, followed by a Monday 13:15 UTC reconciliation run. Each invocation uses the Cloudflare scheduled event time as one stable Effect `DateTime`, scans all Churches, and independently runs idempotent maintenance only for Churches that are due; one Church's failure is captured and reported without failing maintenance for other Churches.

Maintenance eligibility follows the persisted current Cycle `ends_at`, including after a Church Time Zone change, rather than assuming that existing Cycle rows prove rollover occurred. A Church-level checkpoint advances in the same transaction only after rollover, three-Cycle horizon repair, Template Task materialization, Cycle Adjustment application, and Activity writes all succeed. Missing checkpoints are immediately due, allowing the first deployed run to reconcile existing Churches. The design deliberately starts with unbounded Effect concurrency over Hyperdrive, no advisory locks, and no scheduling index; those controls can be introduced if measured scale requires them.
